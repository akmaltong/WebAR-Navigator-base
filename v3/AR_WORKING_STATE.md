# Текущее рабочее состояние проекта WebAR-Navigator (Резервная копия)

В этом файле описано, как именно работают ключевые элементы интерфейса и AR-навигации в текущей стабильной версии (бекапе). Если дальнейшие изменения сломают проект, используйте это описание для восстановления логики.

## 1. Зеленый прицел (Reticle) и вертикальный Гизмо
- **Где находится:** `src/core/WebARSession.ts` (метод `_createReticle`).
- **Структура:** Это `THREE.Group`, в который вложены:
  1. Зеленое кольцо (`RingGeometry`).
  2. Центральная точка (`CircleGeometry`).
  3. Вертикальная линия Гизмо (`BufferGeometry` из двух точек: `(0, 0, 0)` в `(0, 0.10, 0)`). 
- **Параметры:** Длина Гизмо была специально уменьшена до `0.10`, чтобы линия не была слишком длинной и органично смотрелась на поверхностях.
- **Анимация (пульсация):** В `src/main.ts` внутри функции `_onFrame` размер (scale) прицела меняется во времени с помощью синусоиды:
  `const pulse = 1.0 + 0.08 * Math.sin(time / 150);`
  `this.ar.reticle.scale.setScalar(pulse);`

## 2. Белые точки вокруг прицела (FeaturePoints)
- **Где находится:** `src/ar/FeaturePoints.ts`.
- **Логика:** Вместо постоянного создания и удаления партиклов (точек), теперь при загрузке создается **один статичный набор точек (кольца)**. 
- **Структура:**
  - Размер точек (`CircleGeometry`) уменьшен до `0.0075`.
  - Точки расставляются массивом в форме трёх колец: от радиуса `0.05` до `0.09` (всего 36 точек).
- **Анимация:**
  - Группа с точками плавно двигается за основным зеленым прицелом `ringGroup.position.lerp(pos, 0.3)`.
  - Сама группа точек вращается по оси Y (`rotation.y += 0.005`).
  - Прозрачность и размер каждой точки плавно пульсируют в зависимости от времени и угла поворота (интерполяция `Math.sin`).

## 3. UI-интерфейс и панель кнопок (HTML/CSS)
- **Где находится:** `index.html` (стили `#ui-overlay` и `.panel`).
- **Логика позиционирования:**
  - В режиме WebXR DOM Overlay браузер «растягивает» контейнер интерфейса во весь экран.
  - Чтобы карточка со стартовым меню и нижняя панель («Старт», «Маршрут») всегда прилипали ровно к нижней части экрана (независимо от устройства), контейнерам задан Flexbox:
    ```css
    #ui-overlay {
      position: absolute;
      inset: 0;                   /* Растягиваем на весь экран */
      pointer-events: none;       /* Пропускаем клики сквозь контейнер в 3D сцену */
      display: flex;
      flex-direction: column;
      justify-content: flex-end;  /* Прижимаем всё содержимое ВНИЗ */
      padding-bottom: calc(34px + env(safe-area-inset-bottom, 0px));
    }
    .panel {
      pointer-events: auto;       /* Возвращаем кликабельность самим кнопкам */
    }
    ```
  - Стартовое окно `#ar-entry` также центрируется снизу с помощью `justify-content: flex-end` и небольшим отступом `padding-bottom: 50px`.

## 4. Система Hit-Test
- **Где находится:** `src/main.ts` (функция `_onFrame`).
- **Как работает:**
  - Минимальная дистанция луча (`HIT_MIN_DIST`) установлена в `0.15` м. Это позволяет прицелу не пропадать, если пользователь держит камеру близко к полу.
  - При каждом кадре берется позиция поверхности из `hitTestResults`, после чего прицел и белые точки ориентируются по нормали поверхности с помощью кватернионов и `setFromUnitVectors`.
