---

## 📋 Описание проекта

**WebAR Navigator** — мобильное веб-приложение для навигации внутри зданий с использованием дополненной реальности (AR). Пользователь входит в AR-режим через браузер, расставляет точки маршрута на физических поверхностях (пол), и видит анимированный путь в виде шевронов прямо перед собой.

**Целевая платформа:** Android + Chrome  
**Требования:** WebXR Device API, `immersive-ar` сессия, hit-test feature  
**Протокол:** HTTPS (обязательно для WebXR)

## 🔑 Ключевые модули

### `src/main.ts` — ARNavigator
Главный класс-оркестратор. Управляет:
- Инициализацией AR сессии
- Циклом рендеринга (`ar.animate()`)
- **Hit-test фреймворком** — определение позиции ретикла на поверхности
- **State машиной вейпоинтов** — добавление точек в массив
- Построением маршрута через `BezierSpline` и `PathVisualizer`
- Сбросом состояния

**Ключевые константы:**
```typescript
HIT_MIN_DIST = 0.45  // м — минимальная дистанция хита (фильтр "на себя")
FLOOR_MIN_Y  = 0.45  // порог нормали для горизонтальной поверхности
```

**Флоу работы:**
1. Запуск AR → `sessionReady = false`
2. Первый кадр → `_setupHitTest()` — запрашиваем пространства и hit-test source
3. Каждый кадр → `getHitTestResults()` → обновление позиции ретикла
4. Фильтр хитов: игнорируем < 0.45м (тело пользователя)
5. Пользователь нажимает «Добавить точку» → захват `reticle.position`
6. После 2+ точек → кнопка «Маршрут» активна
7. `_buildRoute()` → bezierSpline → PathVisualizer.drawPath()
8. Ретикл скрывается, точки очищаются

---

### `src/core/WebARSession.ts` — WebARSession
Отвечает за инициализацию Three.js и WebXR:
- `THREE.WebGLRenderer` с `xr.enabled = true`
- Сцена, камера, освещение
- AR-кнопка через `ARButton` из Three.js (скрыта визуально, вызывается программно)
- **Ретикл** — маленькое кольцо (R=0.022–0.038м) с центральной точкой, ориентируется по нормали поверхности
- `triggerAREntry()` — программный клик на ARButton

---

### `src/ar/FeaturePoints.ts` — FeaturePoints
Точки трекинга поверхности:
- **Время жизни:** 500мс (эфемерные, не накапливаются)
- **Спавн:** радиальный круг из 16 точек вокруг позиции ретикла
- **На полу:** круг в плоскости XZ (горизонтальный)
- **На стенах:** круг в плоскости XY (вертикальный)
- **Рендеринг:** `THREE.Points` + custom ShaderMaterial
- **Размер:** `22px * scale(dist)` — крупнее, чем стандартные, одинаково видны на разных дистанциях
- **Анимация:** синусоидальное затухание `sin(age * PI)`

---

### `src/ar/PlaneDetection.ts` — PlaneDetection
Классификация поверхностей по нормали hit-test:
- `normal.y > 0.707` → **пол** (floor)
- `normal.y < -0.707` → **потолок** (ceiling)
- иначе → **стена** (wall)
- Хранит карту обнаруженных поверхностей, сливает близкие (radius 0.35м)
- **Визуальные меши отключены** — только логика классификации

---

### `src/ar/PathVisualizer.ts` — PathVisualizer
Визуализация маршрута шевронами:
- Принимает массив `THREE.Vector3[]` — точки bezier-сплайна
- **Геометрия:** каждый шеврон = 2 треугольника левого плеча + 2 треугольника правого = 4 треугольника (заполненная V-форма)
- **Y-координата:** берётся из реальных точек маршрута (floor height из hit-test) + смещение 0.012м
- **Рендеринг:** `BufferGeometry` + `ShaderMaterial`
- **Атрибут `progress`** (0.0–1.0): позиция шеврона вдоль пути (0=старт, 1=финиш)
- **Uniform `uTime`**: время в секундах, обновляется через `requestAnimationFrame`
- **Анимация шейдера:**
  ```glsl
  float wave = sin(progress * 14.0 - uTime * 2.8);
  float alpha = clamp(wave * 0.5 + 0.65, 0.15, 1.0);
  ```
  Волна прозрачности бежит от старта к финишу

---

### `src/utils/BezierSpline.ts` — bezierSpline()
Алгоритм сглаживания пути:
- **Тип:** Smooth Cubic Bezier Spline через N контрольных точек
- **Алгоритм:** Трёхдиагональная система уравнений, метод Thomas (прогонка)
- **C1-непрерывность:** кривая гладко соединяется на стыках сегментов
- **Аналог:** bezier-spline-kotlin (Benedikt Wüller)
- Каждый сегмент — `THREE.CubicBezierCurve3` с 28 сэмплами
- Результат: плотный массив точек для рендеринга

---

### `src/ui/UIManager.ts` — UIManager
Управление интерфейсом:
- `setARActive(bool)` — переключение между оверлеем входа и AR-режимом
- `updateWaypoints(count)` — обновление счётчика точек, цветных индикаторов, текста кнопки
- `setRouteBuilt()` — блокировка кнопок после построения маршрута
- `showStatus(msg, type)` — информационные баннеры (info/success/error)
- `hideLoading()` — скрытие экрана загрузки через 1.7с

## 🎨 UI/UX дизайн

### Цветовая схема
| Цвет | Применение |
|---|---|
| `#00ff88` | Старт, ретикл, основной акцент |
| `#4488ff` | Промежуточные точки маршрута |
| `#ff8800` | Финишная точка |
| `rgba(8,14,26,.9)` | Glassmorphism фон панелей |

### Состояния UI
```
Вход → [AR Entry Overlay]
  ↓ «Запустить AR»
AR активен → [Step Guide] + [Panel]
  Кнопка: 📍 Старт
  ↓ клик (точка 1)
  Кнопка: ➕ Точка 2, 🗺️ Маршрут (disabled)
  ↓ клик (точка 2)
  Кнопки: ➕ Точка 3, 🗺️ Маршрут (enabled)
  ↓ «Маршрут»
  ✅ Маршрут построен → шевроны анимируются
  ↓ «↺ Сброс»
  → Возврат к началу
```

### Маркеры точек
- Сфера (r=0.045м) + кольцо на полу + canvas-спрайт с номером
- 🟢 Точка 1: зелёный (`#00ff88`)
- 🔵 Точки 2..N-1: синий (`#4488ff`)
- 🟠 Последняя точка: оранжевый (`#ff8800`) — окрашивается при построении маршрута

## 🚀 Запуск

### Разработка (локально)
```bash
cd "WebAR Navigator"
npm install
npm run dev
```
Сервер запускается на `https://0.0.0.0:3000`

### Доступ с телефона
```
https://[IP-адрес-компьютера]:3000
```
Например: `https://192.168.1.125:3000`

### Требования к устройству
- Android 8+ 
- Chrome 81+
- ARCore установлен и поддерживается устройством

## 🐛 Известные особенности и ограничения

| Пункт | Описание |
|---|---|
| SSL предупреждение | При первом открытии нужно принять самоподписанный сертификат |
| Только Android | iOS Safari не поддерживает WebXR `immersive-ar` |
| Масштаб ARCore | Точность hit-test зависит от качества сканирования поверхности |
| `local` reference space | Y=0 на высоте устройства при старте сессии, не на полу |
| Граф здания | В `building-graph.json` только примерные данные ФЭСТУ |
